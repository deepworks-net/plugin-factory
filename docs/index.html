Plugin-Factory jQuery Documentation (v1.0.0)

The Plugin-Factory is a lightweight jQuery plugin that quickly and easily adds new plugins to jQuery without the need to worry about replicating standard jQuery plugin boilerplate code. The Plugin-Factory also can extend other plugins added with the factory (similar to class inheritance in OOP languages like Java and C++) and can inherit/override parent functionality.

Behavior:

How the plugin is created:
All plugins must be attached to an element (as jQuery plugins are usually defined). When creating a new plugin, the defaults, framework, user options, and any metadata gathered by the metadata builder functions are extended together. The order in which this occurs is: defaults, then user options, then metadata, then the framework. Keep this in mind when creating a plugin as items last to be added to the plugin will override those that were added first (E.g. a function declared in the defaults will be overridden by a function of the same name that is added in the user options).

Options:

A plugin's options have four main sections: defaults, framework, methods, and builder.
defaults:
This is where the defaults of the plugin are declared. These can be overridden by the user when initializing an existing factory plugin. This is by default, an empty object.
framework:
This is where the behavior of the plugin is defined. This cannot be overridden by a user when initializing an existing factory plugin. This is by default, an empty object. It is suggested to preface framework functions and variables with an underscore to note that it belongs to the framework object but it is not required.
methods:
This is where public methods for the plugin are defined. This cannot be overridden by a user when initializing an existing factory plugin. There are (2) methods pre defined that are available to all factory plugins:
data: Returns: Object
Returns the data object associated with the element the plugin was created on.
api: Returns: Object
Returns the configuration of the plugin associated with the element the plugin was created on.
builder:
This is where initialization methods are defined. This cannot be overridden by a user when initializing an existing factory plugin. There are (2) base methods defined for all factory plugins. Adding either or both of these methods to your plugin options will not override these (2) methods. The behind the scenes builder will take these existing methods and call them first, then additionally call the methods supplied. !!! Talk about what there is access to inside these functions? !!!
metadataFn: Returns: Object
Returns an object with defaults scraped from the element's metadata the plugin is attached to. The initial function returns a blank object. All metadata objects created will be merged together in the order they where added (Base plugin, then all child plugins in the order they were created).
initFn: Returns: Object
Custom Initialize functionality for the plugin. Returns itself. All initialization functions are called in the order they were added (Base plugin, then all child plugins in the order they were created).
